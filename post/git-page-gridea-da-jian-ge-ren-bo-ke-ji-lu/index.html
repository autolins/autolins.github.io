2.实验记录

2-1. 创建一个仓库Token

进入Github，进入Settings/ Developer settings/Personal access tokens ，点击Generate new token。

在New personal access token中，勾选repo的所有权限，最好给这个Token批注用途信息，生成后务必记录下来，一旦网页关闭将无法再次查看此Token。

2-2. 创建一个仓库，存放网站文件

创建一个公开仓库，名为GitHub用户名.github.io。

如何查看GitHub用户名？
登录GitHub，Settings/Account/Change username。

2-3. Gridea远程设置

进入Gridea，点击远程设置，根据步骤2-2的仓库名称和GitHub用户名称和步骤2-1的Token完成配置即可

分支：默认填写master
域名：如果没有购买域名的话，填写步骤2-2的仓库名称即可

点击检测远程链接，出现如下Toast即为成功。

2-4. Gridea同步

点击Gridea中的同步按钮即可。初次部署等待几分钟之后就可以去浏览器访问了，默认地址为你在第 3 步所填写的域名。

出现如下Toast即为成功。

同步成功后，在浏览器输入域名，即可看到初步搭建的博客效果，

同样的，回到步骤2-2在GitHub创建的仓库，也可以发现该仓库也多了许多网站的文件。

至此，使用Gridea搭建博客的实验圆满成功~
